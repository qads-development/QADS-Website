QADS.cloud Error Status Codes Reference

A guide to diagnosing and resolving 4xx Client Errors and 5xx Server Errors encountered on the QADS.cloud hosting platform.

4xx Client Error

The error appears to have been caused by the client (bad request, invalid credentials, or asking for a missing resource).

Code Name Description QADS.cloud Context
400 Bad Request The server cannot process the request due to malformed syntax or invalid parameters. Check your API payload (JSON format, required fields) for structural errors.
401 Unauthorized The request lacks valid authentication credentials. Indicates a missing or invalid API key, token, or session cookie.
403 Forbidden The server understood the request but refuses to authorize access. Check your QADS.cloud ACLs, firewall rules, or application-level user permissions.
404 Not Found The server can't find the requested resource. The URL path is incorrect, or the file/endpoint has been moved or deleted.
405 Method Not Allowed The request method (e.g., POST, PUT) is not supported by the target resource. You might be using a GET request where a POST is required, or vice-versa.
408 Request Timeout The server timed out waiting for the complete client request. Can occur with extremely slow or highly delayed network connections.
410 Gone The resource is no longer available and this condition is considered permanent. Stronger than 404, implying the resource is permanently retired.
413 Payload Too Large The request entity is larger than the server limits allow. File size exceeds the limit set on the QADS.cloud load balancer or application configuration.
415 Unsupported Media Type The request entity has a media type that the server does not support. You might be sending application/xml when the API only accepts application/json. Check your Content-Type header.
429 Too Many Requests The user has sent too many requests in a given amount of time (rate limiting). Your requests have exceeded the established rate limits. Respect the Retry-After header.

5xx Server Error

The server failed to fulfill an apparently valid request. These are issues that must be addressed by the application owner or QADS.cloud support.

Code Name Description QADS.cloud Context
500 Internal Server Error A generic error when an unexpected condition was encountered. CRITICAL. Check your application logs immediately for the root cause (e.g., code exception or database error).
501 Not Implemented The server does not support the functionality required to fulfill the request. Often seen when a client tries to use an unsupported HTTP method (e.g., PATCH).
502 Bad Gateway The server (proxy) received an invalid response from an upstream server. Your application server likely crashed or is unreachable from the QADS.cloud load balancer.
503 Service Unavailable The server is temporarily overloaded or down for maintenance. Check the QADS.cloud status page or your app's resource usage (CPU/Memory).
504 Gateway Timeout The upstream server did not respond within the configured timeout period. Your application took too long to process the request. Optimize the slow operation.
507 Insufficient Storage The server is unable to store the resource needed to complete the request. Your hosting plan's disk space limit or storage quota has been reached.

Troubleshooting & Support

If you encounter persistent errors, please consult your **QADS.cloud logs** first. Provide the **status code**, the **timestamp**, and the **Request ID** when contacting support.
