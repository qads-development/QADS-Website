Frequently asked questions

Have a different question? Reach out to our support team at help@qads.cloud

Why is there no Windows version?

QADS is built entirely in SwiftUI. We utilize native macOS frameworks to render the canvas 10x faster than Electron apps. Porting to Windows would mean rewriting the core engine from scratch without these performance benefits.

Can I use this in the browser?

No. Web browsers cannot access the low-level GPU metal APIs we use for real-time rendering. The web version would lag significantly behind the native experience we aim to provide.

Does it support AI?

Yes. We leverage the Neural Engine (Apple Intelligence) on Apple Silicon for on-device code generation and asset creation. These features require macOS 15+ and are not available on other platforms.

Is it optimized for M1/M2/M3/M4 chips?

Absolutely. The application is compiled specifically for ARM architecture, ensuring minimal battery drain and maximum responsiveness on all Apple Silicon MacBooks. The app can still be ran on older MacBooks.

Do I own the code?

You own it. Publish the code with our own hosting platform, or export to HTML/CSS/JS with the Pro Plan.

Do I need Xcode?

No. The QADS.cloud standalone binary includes everything you need to build and deploy.
